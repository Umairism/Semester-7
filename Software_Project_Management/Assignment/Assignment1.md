# Project Charter

## JWD Consultancy Intranet Website Development Project

### 1. Project Title
JWD Consultancy Intranet Website Development Project

### 2. Project Purpose and Justification
JWD Consultancy currently depends on emails and manual follow-ups for internal communication. This creates delays, duplicate work, and confusion when teams need quick access to updates or shared documents.

This project will deliver a secure intranet portal that gives employees one place to access announcements, share files, communicate internally, and find company resources. The expected result is better coordination, faster information flow, and improved day-to-day productivity.

### 3. Project Objectives
- Design and launch a secure intranet system within **4 months**.
- Provide centralized document upload, storage, and retrieval.
- Improve communication across departments.
- Reduce paperwork and avoid information delays.
- Provide department-wise dashboards for improved task visibility.

### 4. High-Level Requirements
The intranet system must provide:

- Secure user authentication and role-based access.
- Employee dashboard with relevant updates.
- File upload and download functionality.
- Internal announcements and messaging.
- Employee directory.
- Admin control panel for management tasks.

### 5. Project Scope
#### In Scope
- Development of a web-based intranet portal.
- Employee login and role management.
- Document management features.
- Internal communication tools.
- Basic reporting dashboard.

#### Out of Scope
- Public-facing company website updates.
- Mobile application development.
- Customer-facing communication portal.
- Advanced AI-based automation features.

### 6. Stakeholders
- **Project Sponsor:** CEO, JWD Consultancy
- **Project Manager:** Software Team Lead
- **Development Team:** Web developers and designers
- **Testing Team:** QA engineers
- **End Users:** Employees of JWD Consultancy
- **System Administrator:** IT support staff

### 7. Roles and Responsibilities
- **Project Manager:** Oversees planning, schedule, resources, and stakeholder communication.
- **Development Team:** Designs and builds the intranet platform.
- **Testing Team:** Verifies quality, reliability, and usability before release.
- **IT Administrator:** Supports deployment, access management, and maintenance.
- **Project Sponsor:** Reviews progress, approves budget, and signs off on final delivery.

### 10. Assumptions
- Employees have basic computer literacy.
- Required internet and hardware infrastructure is available.
- Management will provide approvals on time.

### 11. Constraints
- Limited project budget.
- Fixed project deadline.
- Small development team.
- Possibility of requirement changes during implementation.

### 12. Risk Management
| Risk | Mitigation Strategy |
| --- | --- |
| Project delay | Track progress weekly and escalate blockers early. |
| Security issues | Implement secure authentication and conduct security testing. |
| Requirement changes | Hold regular stakeholder review meetings. |
| Technical failures | Maintain backups and execute structured test plans. |
